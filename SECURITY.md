# Security Policy

## Supported Versions

Security fixes are prioritized for the latest released version.

## Reporting a Vulnerability

Please do not open a public issue for a suspected security problem.

Instead, report the issue privately through GitHub's security advisory flow for this repository when available. If that is not available, contact the maintainer directly through GitHub.

Include:

- A clear description of the issue.
- Steps to reproduce.
- The affected version or commit.
- Any known workaround.

The goal is to acknowledge reports quickly and coordinate a fix before public disclosure.
